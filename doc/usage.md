# llamune 操作説明書

このドキュメントでは、llamune の基本的な操作方法を説明します。
セットアップがまだの場合は [セットアップ手順](setup.md) を参照してください。

## 1. ログイン

ブラウザで http://localhost:5174 にアクセスします。

セットアップ時に `create_user.py` で作成したユーザー名とパスワードでログインしてください。

管理者ユーザーはすべての機能（プロジェクト・モデル・FTデータ・訓練ジョブの管理）が利用できます。一般ユーザーはチャットのみ利用可能です。

## 2. プロジェクトの作成

サイドバーの「プロジェクト」を開きます。

1. プロジェクト名を入力し「＋ 追加」をクリック
2. 作成されたプロジェクトにシステムプロンプトを設定

システムプロンプトはチャット時に LLM へ送信される指示文です。プロジェクトの用途に合わせて設定してください。

例：
```
必ず日本語で回答してください。
あなたはユーザーの業務を支援するアシスタントです。
```

## 3. モデルの登録

サイドバーの「モデル」を開きます。

### HuggingFace からモデルを検索・登録する

1. 「HuggingFace」タブをクリック
2. 検索ボックスにモデル名を入力して「検索」をクリック（例：`Qwen2.5`）
3. mlx-community の変換済みモデルが一覧表示されます（DL 数も確認できます）
4. 使用したいモデルをクリックするとファイルサイズが表示され、登録フォームに自動入力されます
5. 「＋ 登録」をクリック

> 💡 モデルのファイルサイズ ≒ メモリ使用量です。マシンの搭載メモリに合わせて選択してください。

### 対応モデル

llamune は以下の 2 つのバックエンドを自動判定して使い分けます。

| バックエンド | 対象モデル | 例 |
|---|---|---|
| mlx-lm | テキストモデル | Qwen2.5, Llama, gpt-oss 等 |
| mlx-vlm | マルチモーダルモデル（将来対応予定） | Gemma4 等 |

判定はモデルの `config.json` 内の `model_type` に基づいて自動的に行われるため、ユーザーが意識する必要はありません。

## 4. チャット

サイドバーの「チャット」を開きます。

1. 「モデル選択」で登録済みモデルを選択
2. 「プロジェクト」で使用するプロジェクトを選択
3. メッセージを入力し、`Ctrl+Enter` で送信

初回のモデル選択時はモデルのダウンロードとロードが行われます（モデルサイズによって数分かかる場合があります）。バックエンドのターミナルでダウンロードの進捗を確認できます。

「会話をリセット」ボタンで会話履歴をクリアできます。

## 5. データセット管理

サイドバーの「データセット」を開きます。

RAG 用のドキュメント（知識）を管理する画面です。

### データセットの作成

1. プロジェクトを選択
2. 「データセットを作成」ボタンをクリック

データセット名はプロジェクト名が自動的に使用されます。1 プロジェクトにつき 1 件作成できます。

### ドキュメントの追加

1. 右上の「＋ 追加」ボタンをクリック
2. JSON 形式でドキュメント内容を入力します

```json
{"title": "新宿駅について", "content": "新宿駅は東京都新宿区にある..."}
```

3. 「登録」ボタンをクリック

### ドキュメントの編集・削除

各ドキュメントの右側にある鉛筆アイコンで編集、ゴミ箱アイコンで削除できます。

### データセットの削除

ヘッダー右の赤いゴミ箱ボタンをクリックすると、データセットごとすべてのドキュメントを削除できます。

## 6. FT データ管理

サイドバーの「FT データ」を開きます。

ファインチューニング（FT）に使う会話データを管理する画面です。
会話データは「ベース」と「パターン」の 2 種類に分かれています。

| 種類 | 説明 |
|---|---|
| ベース | 複数パターンで共通する会話の骨格（system + user ターン） |
| パターン | 実際の訓練データ。ベースから派生させて作成するのが基本 |

> 💡 FT データはプロジェクトと紐付いており、使用するモデルのチャットテンプレートに合わせた設計が必要です。1 プロジェクト 1 モデルで運用することを推奨します。

### ベースの作成

1. プロジェクトを選択
2. 編集エリア右上の「ベース」ボタンをクリック
3. `user` ターンに共通の質問文を入力
4. 「保存」をクリック

### パターンの作成

**ベースから派生する場合（推奨）：**

1. **画面を下にスクロール**するとベース一覧が表示されます
2. 対象ベースの右側にある 📋 アイコンをクリック
3. 編集エリアにベースの内容がコピーされた状態になります（タイトルが「新規パターン」になり、ベースIDが表示されます）
4. `assistant` ターンに `[SEARCH]検索キーワード[/SEARCH]` を入力
5. `user` ターンに検索結果として返ってくるドキュメントの内容を入力（ChromaDB の出力形式に合わせる）
6. 2つ目の `assistant` ターンにユーザーへの最終回答を入力
7. 「保存」をクリック

> 💡 ベース一覧は画面下部にあります。編集エリアだけを見ていると気づきにくいのでスクロールして確認してください。

**新規パターンを直接作成する場合：**

1. 編集エリア右上の「パターン」ボタンをクリック
2. 各ターンの内容を入力して「保存」

### train / valid の切り替え

パターン一覧の各行に表示されているバッジ（`train` / `valid`）をクリックすると切り替えられます。

> 💡 `valid` に設定したデータはバリデーション用として訓練ジョブで使用されます。

### JSONL エクスポート

画面右上の「train」「valid」ボタンをクリックすると、各データを `train.jsonl` / `valid.jsonl` としてダウンロードできます。

## 7. 訓練ジョブ

サイドバーの「訓練ジョブ」を開きます。

FT データを使ってベースモデルをファインチューニングする画面です。

### 訓練の実行

1. 「プロジェクト」と「ベースモデル」を選択
2. 訓練パラメーターを設定します

| パラメーター | 説明 | 選択肢 |
|---|---|---|
| max_seq_length | 最大シーケンス長 | 2048 / 4096 / 8192 / 16384 |
| iters | 訓練イテレーション数 | 任意の整数（デフォルト: 100） |
| batch_size | バッチサイズ | 1 / 2 / 4 |
| learning_rate | 学習率 | 1e-05 / 5e-05 / 1e-04 / 5e-04 |

3. 「実行」ボタンをクリック

訓練開始後はジョブが「待機中 → 実行中 → 完了（または失敗）」と遷移します。
画面は 3 秒ごとに自動更新されます。

### ジョブの確認

- ジョブ一覧に各ジョブのステータス・パラメーターが表示されます
- ジョブカードをクリックすると訓練ログが展開表示されます
- 完了したジョブには `adapter_path`（LoRA アダプターの保存先）が表示されます
- Train loss と Val loss が表示されます（Val loss には valid データが必要です）

### 訓練済みモデルのチャット利用

訓練完了後、チャット画面のモデル選択に FT 済みモデル（例：`Qwen2.5-14B FT #10`）が自動登録されます。選択するだけでそのままチャットに使用できます。
